# Validation Artifacts

This directory contains the scripts, result JSON files, figures, report source,
and PDF renderer for the DeepScale downscaling validation report.

## Main Report Files

- `downscaling_validation_report.md`: report source.
- `downscaling_validation_report.pdf`: rendered PDF.
- `render_validation_report_pdf.sh`: Markdown-to-PDF renderer.
- `report_pdf.css`, `report_toc_pages.js`, `fill_report_toc_pages.py`: PDF
  styling and table-of-contents support.

Render the PDF from the repository root with:

```bash
validation/render_validation_report_pdf.sh
```

The renderer uses `pandoc` plus a Chromium-compatible browser such as Google
Chrome or Microsoft Edge. It writes intermediate HTML under
`validation/report_build/`.

## Regenerating Controlled CHIRPS Artifacts

The main controlled CHIRPS result JSON files and report figures can be
regenerated with:

```bash
validation/regenerate_report_artifacts.sh
```

Outputs are written under:

- `validation/results/`
- `validation/figures/`

Use this when you want to refresh the controlled CHIRPS high/low scores, method
reference comparisons, and report-facing figures.

## Environment Variables

- `ROSETTA_SRC`: path to the Rosetta source tree. The regeneration script
  defaults to `../rosetta/src` when that path exists. Set this explicitly when
  using a different Rosetta checkout.
- `PYCPT_IMAGE`: Docker image used for PyCPT/CPT parity runs. Defaults to
  `pycpt:2.10.4`.
- `BROWSER_BIN`: optional path to a Chromium-compatible browser for PDF
  rendering.

Example:

```bash
ROSETTA_SRC=/path/to/rosetta/src \
PYCPT_IMAGE=pycpt:2.10.4 \
validation/regenerate_report_artifacts.sh
```

## PyCPT/CPT Artifacts

The CCA parity comparison uses PyCPT/CPT through Docker. To refresh only the
non-PyCPT artifacts, run:

```bash
validation/regenerate_report_artifacts.sh --skip-pycpt
```

Use the full regeneration command when Docker and the configured PyCPT image
are available.

## Lorenz et al. BCSD Replication

The Lorenz/WDCC BCSD replication is not fully regenerated by the main wrapper
because it depends on local external data:

- WDCC D03 SEAS5-BCSD files.
- Matching Rosetta SEAS5 extracts.
- Matching Rosetta ERA5-Land extracts.

The relevant scripts are:

- `lorenz_bcsd_benchmark.py`
- `lorenz_deepscale_bcsd_compare.py`

These scripts write JSON outputs under `validation/results/` and figures under
`validation/figures/` when the local data are available.

## Report-Facing Locator Maps

The methodology locator maps are generated from locally cached Natural Earth
shapefiles:

```bash
uv run python validation/build_region_locators.py
```

This writes:

- `validation/figures/region_texas_locator_report.png`
- `validation/figures/region_ethiopia_locator_report.png`

## Notes

- The checked-in result artifacts may include historical outputs. For the
  current report, prefer artifacts generated by the current validation scripts.
- The controlled CHIRPS high/low fixtures are implementation checks, not
  operational forecast-skill validation.
